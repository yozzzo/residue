# [A-02] データ読み込み基盤

## 背景
世界/イベントをコード修正なしで追加するため、JSONローダーを先に作る。

## 目的
`data/worlds`, `data/events` の差し替えだけで挙動が変わる状態にする。

## スコープ
- JSONローダークラス実装
- スキーマ最小検証（必須キーのみ）
- ロード失敗時のフォールバック

## 実装タスク
1. `WorldRepository` 実装（一覧取得、ID検索）
2. `EventRepository` 実装（条件付き取得）
3. 必須キー検証（`world_id`, `event_id`, `conditions`, `choices` など）
4. エラー時にログとデフォルトデータを返す

## 完了条件
- JSONを変更するとゲーム内表示が変わる
- 欠損JSONでもクラッシュせずエラー表示で継続

## 検証手順
1. 世界名をJSONで変更
2. 起動後にWorldSelectへ反映されるか確認
3. 必須キーを削除して起動し、クラッシュしないことを確認
